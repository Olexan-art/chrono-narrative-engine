# Source Scoring Settings - Керування Кронами

## 🎯 Огляд

Додано можливість керувати source scoring кронами через Admin Dashboard. Можна вмикати/вимикати як всі крони одразу, так і окремо кожен LLM провайдер.

## 📋 Що додано

### 1. База даних

**Нові поля в таблиці `settings`:**
- `source_scoring_enabled` - головний вимикач для всіх source scoring кронів
- `source_scoring_zai_enabled` - вимикач для Z.AI (кожні 30 хв)
- `source_scoring_gemini_enabled` - вимикач для Gemini (кожну годину о :15)
- `source_scoring_deepseek_enabled` - вимикач для DeepSeek (кожну годину о :30)
- `source_scoring_openai_enabled` - вимикач для OpenAI (кожні 3 години)

Всі поля мають значення `true` за замовчуванням.

### 2. Edge Function

**Автоматична перевірка налаштувань:**
- Перевіряє `source_scoring_enabled` перед виконанням
- Перевіряє налаштування конкретного провайдера
- Повертає `{ success: false, skipped: true }` якщо відключено

### 3. Admin Dashboard

**UI з перемикачами:**
- Головний перемикач "Всі крони" вгорі
- 4 перемикачі для кожного провайдера
- Кольорові індикатори статусу:
  - 🟢 Зелений (пульсуючий) - активний
  - ⚪ Сірий - вимкнено
- Індивідуальні перемикачі неактивні коли головний вимкнено

Прокрутіть до секції "Source Scoring - Якість джерел"

## 💡 Використання

### Вимкнути всі source scoring крони

Переключіть головний перемикач "Всі крони" в положення OFF. Всі провайдери автоматично стануть неактивними (сірі індикатори).

### Вимкнути конкретний провайдер

Натисніть на перемикач поруч з потрібним провайдером. Індикатор стане сірим і крон перестане працювати.

### Тимчасово призупинити оцінювання

1. Вимкніть головний перемикач
2. Крони продовжать спрацьовувати за розкладом
3. Але Edge Function буде пропускати виконання
4. Повертає `{ success: false, error: 'Source scoring is disabled', skipped: true }`

### Причини вимкнення

**Коли вимикати всі крони:**
- Технічне обслуговування
- Досягнуто ліміт API
- Перевірка/налагодження
- Економія коштів API

**Коли вимикати окремий провайдер:**
- Проблеми з конкретним API
- Досягнуто ліміт провайдера
- Тестування інших провайдерів
- Тимчасова недоступність сервісу

## 🔍 Моніторинг

### Перевірка налаштувань через SQL

```sql
SELECT 
  source_scoring_enabled as master,
  source_scoring_zai_enabled as zai,
  source_scoring_gemini_enabled as gemini,
  source_scoring_deepseek_enabled as deepseek,
  source_scoring_openai_enabled as openai
FROM settings;
```

### Перевірка через скрипт

```bash
node check-scoring-settings.mjs
```

Показує:
- Статус головного вимикача
- Статус кожного провайдера
- Розклад запуску
- Кількість активних провайдерів

### Перевірка логів

```bash
node check-llm-logs.mjs
```

Якщо крони вимкнено, в логах побачите записи зі статусом `skipped` або не буде нових записів взагалі.

## ⚙️ Технічні деталі

### Логіка перевірки в Edge Function

```typescript
// 1. Перевірка головного вимикача
if (settings.source_scoring_enabled === false) {
  return { success: false, error: 'Source scoring is disabled', skipped: true };
}

// 2. Визначення провайдера з моделі
const provider = model.includes('glm') ? 'zai' : 
                model.includes('gemini') ? 'gemini' : ...;

// 3. Перевірка налаштування провайдера
if (settings[`source_scoring_${provider}_enabled`] === false) {
  return { success: false, error: `${provider} is disabled`, skipped: true };
}
```

### Поведінка кронів

- Крони pg_cron **продовжують спрацьовувати** за розкладом
- Але Edge Function пропускає виконання якщо вимкнено
- Це дозволяє швидко вмикати/вимикати без зміни розкладу
- Запити до Edge Function = мінімальне споживання ресурсів

### Залежності

- Dashboard використовує React TanStack Query
- Автоматичне оновлення кожні 60 секунд
- Мутація оновлює налаштування миттєво
- Toast повідомлення про успіх/помилку

## 🐛 Можливі проблеми

### Налаштування не зберігаються

**Причина:** Проблема з RLS політиками або ID settings  
**Рішення:**
```sql
-- Перевірити ID
SELECT id FROM settings;

-- Оновити вручну (замініть {id} на справжній ID)
UPDATE settings 
SET source_scoring_enabled = true 
WHERE id = {id};
```

### Edge Function ігнорує налаштування

**Причина:** Старий код не задеплоєно  
**Рішення:**
```bash
npx supabase functions deploy score-news-source --project-ref tuledxqigzufkecztnlo
```

### Крони працюють незважаючи на вимикач

**Причина:** Кешування або затримка  
**Рішення:**
- Зачекати 1-2 хвилини
- Перевірити логи Edge Function
- Перезапустити крон вручну для тесту

## 📊 Статистика використання

Після застосування перевірте дашборд:
- Кольорові індикатори показують стан у реальному часі
- "Останні 10 оцінок" - моніторинг результатів
- Якщо немає нових оцінок і всі індикатори сірі - налаштування працюють
